Architecture

"A constantly-running rewind buffer for your digital life." You forget what you just saw → hit a hotkey → scroll backward → recover Screen —⌋ Audio —⌋—⇒ [ ring buffer ] —⇒ rewind —⇒ recover Context—⌋ always recording on demand text/links only saving when you need it

1. Capture

Screen frames via ScreenCaptureKit
Audio via AVAudioEngine (mic + system)
Metadata: active app, window title

2. Buffer

Rolling ring buffer (5-60 min)
Old data overwritten continuously
Fixed budget: size(buf) ≤ K

3. Index

OCR: Vision framework → FTS5
Transcription: Speech framework
Metadata: NSWorkspace events

4. Retrieve

Timeline UI — scroll back in time
Full-text search across all sources
Export clips on demand

A. Screen Pipeline Screen → ScreenCaptureKit → Frame Stream → Encoder → Ring Buffer B. Audio Pipeline Mic/System → AVAudioEngine → AAC Encoder → Ring Buffer C. Metadata Pipeline NSWorkspace / Accessibility API → Event Stream → SQLite FTS5 Ring Buffer [ t-60 ] [ t-59 ] ... [ t-1 ] [ NOW ] ↑ overwritten continuously Fixed disk budget · Constant-time writes · Crash-safe chunks

Screen Capture

ScreenCaptureKit (macOS 13+)
Hardware encode: VideoToolbox → H.264
2 fps background / 30 fps review

Audio Capture

AVAudioEngine tap on inputNode
AVAudioPCMBuffer → AAC → m4a
Auto-rotate on chunk duration

Storage

Circular chunk files on disk
Configurable budget (100MB-5GB)
~/Library/Application Support/Murmur/

OCR

Vision: VNRecognizeTextRequest
Rate-limited async, never blocks capture
Results → SQLite FTS5

Transcription

Speech framework, on-device
Lazy: runs behind capture pipeline
Transcripts → SQLite FTS5

Search

SQLite + FTS5
OCR text + transcripts + metadata
Ranked results across all sources
